**状态**：已完成

## 1. 项目概述

### 1.1 项目背景

短视频已成为信息传播的主流形式，知识科普类内容在抖音、快手、视频号等平台有大量受众。然而，高质量短视频的制作门槛较高，涉及脚本策划、配音录制、画面制作、字幕匹配、后期剪辑等多个环节，人力成本高、制作周期长。

本项目旨在利用 AI 大模型技术，构建一条全自动的短视频生产线，用户只需提供主题即可获得完整的 MP4 视频文件。

### 1.2 项目目标

1. 输入一个主题，自动输出 45-90 秒的知识科普类竖屏短视频
2. 全流程自动化，无需人工干预
3. 支持阶段缓存，可断点续跑
4. 支持自定义脚本要求和背景音乐

### 1.3 适用范围

- 目标用户：内容创作者、自媒体运营者、教育从业者
- 视频类型：知识科普类（科学、技术、历史、文化等）
- 发布平台：抖音、快手、小红书、视频号等竖屏短视频平台

## 2. 功能需求

### 2.1 功能列表

| 编号 | 功能模块 | 优先级 | 描述 |
|------|---------|--------|------|
| F01 | 脚本生成 | P0 | 根据主题，调用 DeepSeek 生成结构化脚本（标题+分镜+旁白+画面描述） |
| F02 | 配音合成 | P0 | 将脚本旁白合成为自然语音（优先 MiniMax TTS，回退 Edge TTS） |
| F03 | 配图生成 | P0 | 根据分镜描述调用 MiniMax 文生图生成 9:16 竖屏图片 |
| F04 | 字幕生成 | P0 | 对配音进行语音识别生成 SRT 字幕 |
| F05 | 视频合成 | P0 | 合成最终视频（Ken Burns 动效 + 字幕 + 配音 + BGM） |
| F06 | 开场/片尾 | P1 | 自动添加标题画面和引导关注片尾 |
| F07 | BGM 混音 | P1 | 支持本地音乐文件混音 |
| F08 | 缓存复用 | P1 | 各阶段输出缓存，可跳过已完成的步骤 |
| F09 | 音乐自动生成 | P2 | 调用 MiniMax 音乐生成 API 自动创建 BGM |

### 2.2 用户交互流程

```
用户输入 topic 和可选参数
    │
    ▼
┌─ ① 脚本已缓存？──────────┐
│ 是 → 读取缓存              │
│ 否 → DeepSeek 生成 → 缓存  │
└─────────┬─────────────────┘
          ▼
┌─ ② 配音已缓存？──────────┐
│ 是 → 读取缓存              │
│ 否 → MiniMax/Edge TTS → 缓存│
└─────────┬─────────────────┘
          ▼
┌─ ③ 配图已缓存？──────────┐
│ 是 → 读取缓存              │
│ 否 → MiniMax 文生图 → 缓存 │
└─────────┬─────────────────┘
          ▼
┌─ ④ 字幕已缓存？──────────┐
│ 是 → 读取缓存              │
│ 否 → Whisper/预估 → 缓存   │
└─────────┬─────────────────┘
          ▼
┌─ ⑤ 视频合成 ─────────────┐
│ 图片 + 音频 + 字幕 + BGM   │
│ moviepy → MP4              │
└─────────┬─────────────────┘
          ▼
       输出 MP4
```

## 3. 接口需求

### 3.1 DeepSeek API

- **用途**：脚本生成
- **协议**：HTTP REST，OpenAI 兼容格式
- **模型**：`deepseek-chat`
- **端点**：`https://api.deepseek.com/v1/chat/completions`
- **输入**：主题 + 可选的额外要求
- **输出**：结构化 JSON（标题、分镜、旁白、画面描述、BGM 风格）

### 3.2 MiniMax API（文生图）

- **用途**：配图生成
- **协议**：HTTP REST
- **模型**：`image-01`
- **端点**：`POST https://api.minimaxi.com/v1/image_generation`
- **认证**：Bearer Token + Group ID
- **输出**：图片下载 URL 列表
- **比例**：9:16（竖屏）

### 3.3 MiniMax API（TTS）— 可选

- **用途**：语音合成（当账户有余额时使用）
- **模型**：`speech-02`
- **端点**：`POST https://api.minimaxi.com/v1/t2a_pro`
- **输出**：MP3 音频数据

### 3.4 Edge TTS（免费备用）

- **用途**：语音合成（当 MiniMax TTS 不可用时使用）
- **协议**：WebSocket（通过 edge-tts Python 库）
- **默认音色**：`zh-CN-XiaoxiaoNeural`（中文女声）

### 3.5 faster-whisper（ASR，可选）

- **用途**：配音转文字获取时间戳
- **类型**：本地模型，需下载
- **模型大小**：base（约 150MB）
- **回退方案**：按文本长度比例分配时间

## 4. 数据需求

### 4.1 脚本数据结构

```json
{
  "title": "视频标题",
  "bgm_style": "背景音乐风格描述",
  "scenes": [
    {
      "narration": "旁白文字（30-60字）",
      "image_prompt": "画面描述提示词",
      "keywords": ["关键词1", "关键词2"]
    }
  ],
  "outro_text": "片尾引导文案"
}
```

### 4.2 配置文件

```ini
# .env
DEEPSEEK_API_KEY=sk-xxx
MINIMAX_API_KEY=sk-xxx
MINIMAX_GROUP_ID=xxx
```

### 4.3 视频输出规格

| 参数 | 值 |
|------|-----|
| 分辨率 | 1080×1920（竖屏） |
| 帧率 | 30fps |
| 视频编码 | H.264 |
| 音频编码 | AAC |
| 码率 | ~5000kbps |
| 文件格式 | MP4 |
| 目标时长 | 45-90 秒 |

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 端到端耗时 | < 15 分钟 | 含 API 调用 + 视频渲染，5 分镜场景 |
| API 调用间隔 | ≥ 1 秒 | 避免 MiniMax 限流 |
| 渲染并发 | 1 线程 | 当前为串行处理 |

### 5.2 可用性

- 各阶段有缓存，网络/API 故障后可续跑
- 无 GPU 环境下自动使用 CPU 模式
- TTS 有双重回退机制：MiniMax → Edge TTS → 报错提示

### 5.3 可维护性

- 配置集中管理（config.py）
- 各模块独立，可单独测试和替换
- 清晰的日志输出（运行状态 + 耗时）

## 7. 验收标准

| 编号 | 验收项 | 验证方法 |
|------|--------|---------|
| A01 | 输入主题后生成完整 MP4 | 运行 `python pipeline.py --topic "测试"` |
| A02 | 视频分辨率正确 | `ffprobe` 检查输出 |
| A03 | 配音清晰可听 | 人工检查 |
| A04 | 配图与内容相关 | 人工检查 |
| A05 | 字幕与配音同步 | 人工检查 |
| A06 | 缓存复用正确 | 两次运行同一主题，第二次跳过已有步骤 |
| A07 | TTS 回退正常 | 清空 MiniMax 余额，确认自动使用 Edge TTS |
| A08 | 错误处理 | 网络断开时给出清晰提示 |
